# tdx-api 旧工具替代差距分析

日期：2026-06-27

## 分析口径

- 已经实现：`tdx-api` 当前已有直接接口或已经完成的 agent 聚合接口。
- 可以实现：通达信底层已有相近数据，只需要补 handler、聚合方法或中文 text 输出。
- 无法替代：数据源口径不属于通达信行情/财务/F10能力，建议保留外部源。
- anysearch 单列：`anysearch` 本来就是协同搜索源，不算 `tdx-api` 替代失败，只列目标口径。

## 已经实现

| 旧工具能力/数据类型 | 旧来源 | 当前 tdx-api 对应能力 | 说明 |
|---|---|---|---|
| 个股实时行情 | 腾讯 `qt.gtimg.cn`、新浪行情 | `/api/quote`、`/api/agent/stock-brief`、`/api/agent/stock-brief-text` | 已覆盖价格、涨跌幅、开高低、昨收、成交量、成交额等核心行情。 |
| 日 K 原始数据 | 腾讯 K 线、Baostock 备选 | `/api/kline`、`/api/kline/all`、`/api/kline/qfq`、`/api/kline/hfq` | 已覆盖 OHLCV、复权 K 线。 |
| 周线/月线聚合 K 线 | 旧脚本计算或第三方 K 线 | `/api/agent/kline-summary` | 已按日/周/月输出原始聚合数据。 |
| K 线中文摘要 | 旧 agent 自行分析 K 线 | `/api/agent/kline-summary-text` | 已输出低噪音中文摘要，适合 agent 直接阅读。 |
| MA、MACD、RSI、BOLL、ATR | `technical_indicators.py` | `/api/agent/technical-summary` | 已覆盖主要趋势、动量、波动指标。 |
| 基础财务数据 | 东方财富 F10、通达信财务 | `/api/finance`、`/api/agent/stock-brief` | 已覆盖股本、资产、净资产、营收、利润、现金流、股东数等基础项。 |
| F10 原文/分类内容 | 东方财富 F10、通达信 F10 | `/api/f10` | 已具备 F10 分类读取能力，可供后续解析。 |
| 股票名称/代码检索 | 旧脚本硬编码或外部列表 | `/api/search`、`/api/stocks/search`、`/api/codes` | 已支持基础代码与名称检索。 |
| 所属板块 | 东方财富概念/行业 | `/api/blocks`、`/api/hy`、`/api/agent/stock-brief` | 已有板块归属查询与 stock brief 聚合。 |
| 指数 K 线 | 腾讯/新浪/ TickFlow 指数 | `/api/index/kline`、`/api/index/all` | 可替代主要指数 K 线类数据。 |
| 52 周高低、成交额变化类统计 | 东方财富/行情衍生统计 | `/api/stat`、`/api/moneyflow`、`/api/agent/stock-brief` | 当前已有粗粒度统计字段。 |

## 可以实现

| 旧工具能力/数据类型 | 旧来源 | tdx-api 可实现方式 | 建议优先级 |
|---|---|---|---|
| 市场概况快照 | `market_context_daily.py`：主要指数 + 持仓股 | 基于 `/api/index/kline`、`/api/quote` 做 `market-context` 聚合接口 | 高 |
| 持仓股批量行情 | 腾讯批量行情 | 增加批量 quote 聚合，或让 agent 循环调用 `/api/quote` | 高 |
| KDJ | `technical_indicators.py` | 复用 K 线，在 `/api/agent/technical-summary` 中补计算 | 高 |
| BIAS | `technical_indicators.py` | 复用 MA 与收盘价，在 technical summary 补计算 | 高 |
| 技术总分/多空评分 | `technical_indicators.py` | 在已有 MA/MACD/RSI/BOLL/ATR/KDJ/BIAS 上做规则评分 | 中 |
| 成交量评分 | `technical_indicators.py` | 基于近 N 日均量、量比、价量关系聚合 | 中 |
| PE/PB/换手率/市值聚合 | Baostock、腾讯、东方财富 | 财务 + quote + 股本字段可组合实现，部分字段需确认底层 quote 是否稳定返回 | 中 |
| 全市场 TopN/涨跌分布 | 东方财富全市场行情 | 基于 `/api/codes` + `/api/quote` 批量扫描实现，注意性能与缓存 | 中 |
| 概念/行业板块强弱排名 | AKShare/东方财富板块 | 基于板块成分 + 个股行情聚合；若缺成分表，需要先补板块数据入库 | 中 |
| 个股资金流粗粒度 | AKShare/东方财富资金流 | 现有 `/api/moneyflow` 可做弱替代；分档资金流不能严格等价 | 中 |
| F10 财报字段中文化 | 东方财富 datacenter F10 | 已并入 `/api/agent/stock-brief-text`；深度 F10 资料由 `/api/agent/f10-summary-text` 补充 | 中 |
| 股东结构粗解析 | 东方财富股东接口 | 若通达信 F10 章节含股东文本，可解析为弱结构化字段 | 低 |
| 公告关键词筛选 | 东方财富公告 API | `/api/f10?cat=11` 可做公告章节弱替代，适合标题/关键词摘要 | 低 |
| 非交易日市场快照文件输出 | 旧脚本写 JSON 文件 | 可新增 agent 调用脚本或缓存接口；不是核心行情能力 | 低 |

## 无法替代

| 旧工具能力/数据类型 | 旧来源 | 为什么不能由 tdx-api 严格替代 | 建议 |
|---|---|---|---|
| 财联社实时新闻流 | `www.cls.cn/api/cache?name=telegraph` | 通达信行情/F10不提供实时新闻流口径 | 保留外部新闻源或改由 anysearch 补充。 |
| 东方财富公告 API 的最新公告列表与 `art_code` | `np-anotice-stock.eastmoney.com` | 通达信 F10 可读公告章节，但不等价于东方财富公告列表元数据 | 需要最新公告列表时保留东方财富公告 API。 |
| 东方财富 datacenter 结构化股东字段 | `RPT_F10_EH_HOLDERS` | 通达信 F10 可能有文本，但缺稳定结构化字段口径 | 若要机构类型、持股比例变动，继续保留东方财富 datacenter。 |
| 东方财富 push2his 分档资金流 | 超大单/大单/中单/小单 | 当前 tdx-api 只具备粗粒度统计/资金流，不能还原分档口径 | 保留外部源；tdx-api 只做弱替代摘要。 |
| 外部平台概念热度榜 | AKShare/东方财富概念榜 | 平台榜单口径来自外部排序，不是通达信原生行情字段 | 可保留外部榜单，或用 tdx-api 自算“内部强弱榜”。 |
| TickFlow/Tushare/Baostock 作为独立备源 | 多个第三方数据源 | 备源价值在于交叉验证和容灾，不是单一 tdx-api 能替代的能力 | 若要求数据冗余，仍需保留至少一个外部备源。 |

## anysearch 协同资讯目标口径

| 目标口径 | 查询目标 | 给 agent 的用途 | 是否并入 tdx-api |
|---|---|---|---|
| 公司近期催化 | 公司名 + 股票代码 + 最近公告/订单/中标/扩产/回购/减持 | 解释股价异动、补充短线事件驱动 | 否，作为外部搜索上下文。 |
| 产业链位置 | 公司名 + 主营产品 + 上下游 + 客户/供应商 | 判断公司处于产业链哪个环节、受哪些价格和需求影响 | 否。 |
| 行业政策 | 行业关键词 + 最新政策 + 补贴/监管/招标/出口限制 | 判断政策顺风或逆风 | 否。 |
| 海内外可比公司 | 公司名 + 产品 + competitor/peer | 给估值和景气度找参照物 | 否。 |
| 全球映射 | A 股公司主营 + 海外需求/海外项目/海外政策 | 判断外需、汇率、海外订单影响 | 否。 |
| 舆情与风险 | 公司名 + 处罚/诉讼/质量/安全/债务/商誉 | 补充非财务风险点 | 否。 |
| 研报观点交叉验证 | 公司名 + 券商研报 + 业绩预测 + 目标价 | 只作为观点参考，不作为确定事实 | 否。 |
| 新闻时间线 | 公司名 + 最近 30/90 天新闻 | 帮 agent 建立事件顺序，避免只看单条消息 | 否。 |

## 最小实施顺序

1. 先补 `/api/agent/technical-summary`：KDJ、BIAS、技术总分、成交量评分。
2. 再做 `market-context`：主要指数 + 持仓股批量行情 + 简短中文摘要。
3. 财务字段已并入 `stock-brief-text`；深度资料继续完善 `f10-summary-text` 的分类裁剪。
4. 新闻、政策、产业链、全球映射继续交给 `anysearch`，不要硬塞进 `tdx-api`。
